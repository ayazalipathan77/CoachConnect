# Deploying CoachConnect to Render

This repo ships a Render **Blueprint** (`render.yaml`) that provisions everything
in one go: a managed **Postgres 16** database + a **Node web service** running the
Next.js app. Migrations and demo seeds run automatically during the build.

## One-time manual deploy (≈3 minutes of clicking)

1. Go to **https://dashboard.render.com** and sign in.
2. **New → Blueprint**.
3. **Connect** the GitHub repo `ayazalipathan77/CoachConnect` (authorize Render
   for GitHub if asked). Render reads `render.yaml` from the **main** branch.
4. Render shows the plan: a database `coachconnect-db` + a web service
   `coachconnect`. Click **Apply**.
5. Wait for the first build. The build:
   - installs deps
   - applies DB migrations (must happen before the build, since some pages
     query tables that need to exist at build time)
   - runs `next build`
   - runs `pnpm db:deploy` → re-applies migrations (no-op) + seeds 20 sports,
     8 demo coaches, an admin account, and default platform settings
6. When the web service is **Live**, open its `*.onrender.com` URL.

That's it — no environment variables to set by hand. `AUTH_SECRET` is generated by
Render; `DATABASE_URL` is wired from the database; integrations default to `mock`.

## What you get

- Public marketing site + About page
- Sign up / log in (athlete or coach), sessions, dashboards
- Discover coaches → coach profile → book a slot → mock escrow payment → My Bookings

## Notes & next steps

- **Free tier:** the free web service sleeps after inactivity (cold starts ~30s)
  and the free Postgres is time-limited — fine for a demo, upgrade for production.
- **Region:** set to `frankfurt` (EU/UK) in `render.yaml`; change if you prefer.
- **Real integrations:** flip the `INTEGRATION_*` env vars from `mock` to `real`
  and add the matching keys (Stripe, Maps, Email, Push) in the Render dashboard.
- **Custom domain / Cloudinary / OAuth:** add the relevant env vars in the
  service's *Environment* tab and redeploy.

## Local development

```bash
docker start coachconnect-db          # Postgres 16 on port 5440
pnpm install
pnpm db:migrate && pnpm db:seed && pnpm db:seed:coaches
pnpm dev                              # http://localhost:3000
```
